# 开发路线图

## 愿景

构建一个高性能、轻量级、AI 友好的 Mermaid 图表命令行工具，为开发者和 AI 系统提供一个可靠的图表生成基础设施。

## 版本规划

```
v0.1 (MVP)              v0.2 (扩展)           v1.0 (稳定)
├── 基础框架             ├── 更多图表类型     ├── 完整功能集
├── Flowchart           ├── 高级样式         ├── 性能优化
├── Sequence            ├── 主题系统         ├── 文档完善
├── 错误纠错             ├── PNG 导出         ├── 社区驱动
└── CLI 接口            └── 性能优化         └── 长期维护
```

---

## 阶段 1: P1 - 基础设施（1-2 周末）

**状态**：未开始  
**目标**：建立项目框架，完成 Flowchart 基础解析和 SVG 生成

### 任务清单

- [ ] 初始化 Cargo 项目
  - 设置依赖：`clap`, `serde`, `serde_json`
  - 配置 Cargo.toml
  - 创建 README 和基本文档

- [ ] 实现词法分析器（Lexer）
  - 识别基础关键字：`graph`, `flowchart`, `TD`, `LR` 等
  - 识别节点定义：`A[label]`, `B{diamond}` 等
  - 识别连接符：`-->`, `--`, `->` 等
  - 跟踪行列号用于错误诊断

- [ ] 定义 AST 结构
  - `Diagram` — 图表根节点
  - `Node` — 节点信息（id, label, shape）
  - `Edge` — 边信息（from, to, label, type）
  - `NodeShape` — 节点形状枚举

- [ ] 实现语法解析器（Parser）
  - 解析图表方向声明
  - 解析节点定义和边定义
  - 基础错误检测（缺失关键字、格式错误）
  - 错误恢复（继续解析以报告多个错误）

- [ ] 实现基础 SVG 生成
  - 简单的矩形节点渲染
  - 直线连接箭头
  - 节点标签显示
  - 固定尺寸（不进行自动布局）

- [ ] 单元测试
  - Lexer 测试（20+ 用例）
  - Parser 测试（15+ 用例）
  - SVG 生成测试（10+ 用例）
  - 目标覆盖率：> 70%

### 验收标准

```bash
# 基础流程图渲染
$ echo 'graph TD; A[Start]-->B[End]' | ./mermaid-cli --stdin -o test.svg
# ✅ 生成有效的 SVG 文件

$ ./mermaid-cli example.mmd -o output.svg
# ✅ 从文件读取和输出

# 错误处理
$ echo 'grpah TD' | ./mermaid-cli --stdin
# ✅ 提示语法错误
```

### 时间估计
- **工时**：2-3 个人天（周末）
- **日历时间**：1-2 周末

## 阶段 2: P2 - MVP 发布（2-3 周末）

**状态**：等待 P1 完成  
**目标**：完整的 Flowchart 支持 + 错误纠错，发布 v0.1-alpha

### 新增功能

#### 2.1 Flowchart 功能完善

- [ ] 扩展节点形状支持
  - `[]` — 矩形
  - `()` — 椭圆
  - `{}` — 菱形
  - `[()]` — 圆形矩形
  - `[[]]` — 矩形框
  - `[(diamond)]` — 命名菱形

- [ ] 样式支持
  - 节点颜色 `style A fill:#f9f,stroke:#333,stroke-width:2px`
  - 边标签和样式
  - 基础字体控制

- [ ] 高级特性
  - 条件分支 `A -->|if| B` 和 `A -->|else| C`
  - 子图（subgraph）
  - 多行标签

#### 2.2 自动错误修复

- [ ] 实现修复器（Fixer）
  - 缺失 `end` 自动添加
  - 缺失或不匹配的引号补全
  - 常见拼写错误识别
    - `grpah` → `graph`
    - `-->>` → `-->`
    - `=>` → `->`

- [ ] 诊断信息格式
  - 位置信息（行号、列号）
  - 错误分类（SyntaxError, TypoError 等）
  - 修复建议和置信度

- [ ] CLI 命令扩展
  - `check` — 检查语法
  - `check --show-fixes` — 显示修复建议
  - `fix` — 自动修复并输出

#### 2.3 改进渲染

- [ ] 自动布局
  - 简单的层级布局（从上到下）
  - 节点间距和对齐
  - 边的路由优化

- [ ] 视觉改进
  - 节点形状精确渲染
  - 边标签位置
  - 更好的字体和颜色

#### 2.4 测试和质量

- [ ] 集成测试
  - 完整工作流测试
  - 边界情况和错误恢复
  - 性能基准测试

- [ ] 错误处理
  - 无效输入处理
  - 文件 I/O 错误
  - 编码问题（UTF-8 支持）

- [ ] 代码覆盖率
  - 目标 > 80%
  - 关键路径 100% 覆盖

#### 2.5 发布准备

- [ ] 跨平台编译
  - Linux x86_64
  - macOS x86_64 和 ARM64
  - Windows x86_64

- [ ] GitHub Actions CI/CD
  - 自动编译二进制
  - 运行测试
  - 发布 Release

- [ ] 文档完善
  - README 更新
  - 示例和用例
  - 故障排除指南

### 验收标准

```bash
# 完整的 Flowchart 支持
$ mermaid-cli complex-diagram.mmd -o output.svg
# ✅ 支持多种节点形状、样式、子图

# 错误纠错
$ echo 'grpah TD\nA-->B' | ./mermaid-cli --stdin --show-fixes
# ✅ 返回修复建议

$ ./mermaid-cli broken.mmd --fix -o fixed.mmd
# ✅ 自动修复文件

# 性能
$ time ./mermaid-cli large-diagram.mmd -o output.svg
# ✅ 在 100ms 内完成（< Node.js 版本的 1/5）

# 可用性
$ ./mermaid-cli --help
# ✅ 清晰的帮助信息和使用示例
```

### 时间估计
- **工时**：4-5 个人天
- **日历时间**：2-3 周末
- **发布时间**：第 5-8 周

## 阶段 3: P3 - 扩展和优化（3-4 周末）

**状态**：规划中  
**目标**：添加序列图支持，优化性能和用户体验

### 新增功能

#### 3.1 序列图支持

- [ ] 实现序列图解析
  - 参与者（participant）定义
  - 消息类型：`->`, `-->`, `-x`, `-->>`, `->>` 等
  - 消息标签

- [ ] 序列图渲染
  - 参与者水平排列
  - 时间线垂直显示
  - 消息箭头和标签
  - 激活框（activation boxes）
  - 循环、选择、并行控制结构

- [ ] 序列图纠错
  - 参与者定义验证
  - 消息端点检查
  - 常见错误自动修复

#### 3.2 用户体验改进

- [ ] 配置选项扩展
  - `--width`, `--height` — 输出尺寸
  - `--theme dark|light` — 主题选择（预留）
  - `--no-background` — 透明背景
  - `--format json` — JSON 输出诊断

- [ ] 性能优化
  - 大型图表处理（1000+ 节点）
  - 内存使用优化
  - 并行处理（如果适用）

- [ ] 更好的错误消息
  - 颜色化输出（支持终端颜色）
  - 错误上下文显示（周围几行代码）
  - 建议的修复高亮

#### 3.3 高级功能

- [ ] 样式系统初步实现
  - 节点和边的基础样式类（class）
  - 内置样式预设
  - 自定义样式文件支持（考虑）

- [ ] 导出改进
  - SVG 元数据支持（便于后续编辑）
  - 多页输出（大图表分页）

### 验收标准

```bash
# 序列图支持
$ mermaid-cli sequence.mmd -o output.svg
# ✅ 支持序列图渲染和纠错

# 配置选项
$ mermaid-cli diagram.mmd -o output.svg --width 1024 --height 768
# ✅ 尺寸配置生效

# 性能
$ time mermaid-cli large-1000-nodes.mmd -o output.svg
# ✅ 在 500ms 内完成

# 诊断信息
$ mermaid-cli broken.mmd --format json
# ✅ 返回结构化的诊断 JSON
```

## 依赖和风险

### 潜在阻碍

1. **复杂的布局算法** — Flowchart 自动布局可能需要更复杂的算法
   - **解决方案**：MVP 使用简单布局，P3+ 优化

2. **跨平台编译** — Windows 编译可能遇到依赖问题
   - **解决方案**：早期设置 CI/CD，跨平台测试

3. **性能瓶颈** — 大型图表可能有性能问题
   - **解决方案**：使用基准测试，及时优化

### 依赖管理

- 最小化外部依赖（只用必需的 crate）
- 定期审计依赖的安全性
- 保持 Rust 版本最新
